GetValidOutputPath will try to combine output path based on 3 possible patterns. If the resulting path is valid it will return the new path and null as the Exception.

The default naming convention of the output file will be as {outputType}_{–name ?? "untitled"}_{yyyyMMdd}_{HHmmss}.{type} There are 3 possible input formats we can deal with, we will use the default format alias as {DEF}

  • 1 User does not provide –out ./{DEF}
  • 2 User provides both –name and directory path {–out}/{DEF}
  • 3 User provides full path with filename for –out {–out}
Parameters

fileNameFile name –name
Returns
A tuple of string and Exception go-style. A new output path will be returned when valid and null otherwise, vice-versa for exception.
